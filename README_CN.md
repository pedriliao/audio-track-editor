# 后端音频轨道编辑器

使用 C 语言实现的后端音频轨道编辑引擎，采用共享内存和层级数据结构实现高效的音频片段管理。

## 概述

本项目实现了一个后端音频编辑引擎，支持以下功能：
- **WAV 文件读写** — 加载和保存 16 位 PCM 单声道音频
- **轨道操作** — 基于链表的轨道段，支持读取、写入和删除音频采样
- **广告识别** — 基于互相关的音频模式检测，用于识别音频轨道中的广告
- **复杂插入** — 使用共享后备存储的逻辑插入，通过父子关系最小化内存使用
- **进程间共享内存模型** — 引用计数的共享块实现多轨道编辑中的零拷贝数据共享

## 架构

### 数据结构

```
sound_seg (轨道)
 └── track_node (链表) ──→ track_node ──→ track_node ──→ ...
      └── shared_block              └── shared_block (可共享)
           └── int16_t[] data            └── ref_count 追踪所有权
```

- **`sound_seg`** — 表示一个完整的音频轨道，由链表节点组成
- **`track_node`** — 引用共享块中连续范围的段。通过 `pa_or_ch` 字段追踪关系类型：父节点 (0)、子节点 (1) 或普通节点 (2)
- **`shared_block`** — 引用计数的采样数据。多个节点可共享同一块，实现 `tr_insert()` 的写时引用语义

### 核心设计决策

1. **链表分段** — 允许高效的插入和删除操作，无需移动数据
2. **共享后备存储** — `tr_insert()` 通过共享 `shared_block` 指针创建父子关系，插入数据在内存中仅存在一份
3. **引用计数** — `shared_block.ref_count` 确保仅在无节点引用时才释放内存
4. **写穿共享** — 对子节点部分的写入通过共享块立即在父节点中可见（反之亦然）

## API 参考

### 轨道生命周期
| 函数 | 描述 |
|------|------|
| `tr_init()` | 分配一个空轨道 |
| `tr_destroy(track)` | 释放所有内存（遵循引用计数） |

### 基本操作
| 函数 | 描述 |
|------|------|
| `tr_length(track)` | 返回采样数 |
| `tr_read(track, dest, pos, len)` | 将采样复制到缓冲区 |
| `tr_write(track, src, pos, len)` | 写入采样（自动扩展） |
| `tr_delete_range(track, pos, len)` | 删除采样；如果父节点有活跃子节点则返回 `false` |

### 广告识别
| 函数 | 描述 |
|------|------|
| `tr_identify(target, ad)` | 通过互相关查找不重叠的广告出现位置（95% 阈值） |

### 复杂插入
| 函数 | 描述 |
|------|------|
| `tr_insert(src, dest, destpos, srcpos, len)` | 使用共享后备存储的逻辑插入 |

## 构建

```bash
make              # 构建 sound_seg.o
make clean        # 清除构建产物
```

使用 `-Wall -Werror -fPIC -Wvla -g` 编译。输出为目标文件 (`sound_seg.o`)，设计为链接为共享库使用。

## 技术亮点

- **零拷贝插入**：`tr_insert()` 在源和目标之间共享底层数据块，将每次插入的内存使用从 O(n) 降至 O(1)
- **互相关检测**：`tr_identify()` 使用归一化互相关检测相似度 ≥ 95% 的广告模式
- **正确的父子生命周期**：删除操作正确处理级联引用计数更新，包括自插入后子节点先于父节点出现在链表中的边界情况
- **无禁用函数**：避免使用 `printf`（内部使用动态内存）、变长数组和受限系统调用
